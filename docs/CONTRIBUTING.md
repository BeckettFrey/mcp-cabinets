# Contributing to MCP cabinets

Thank you for your interest in contributing! Your help is welcome to improve MCP cabinets as a modular knowledge bank.

## How to Contribute

- **Report Issues:** Use GitHub Issues to report bugs, suggest features, or request documentation improvements.
- **Submit Pull Requests:** Fork the repository, create a feature branch, and submit a pull request with a clear description of your changes.
- **Documentation:** Help improve or expand documentation in the `/docs` folder.
- **Code Quality:** Follow existing code style and add tests for new features or bug fixes.

## Guidelines

- Keep changes focused and atomic.
- Reference related issues in your pull request.
- Update relevant documentation when making changes.
- Be respectful and collaborative in discussions.

## Getting Started

1. Clone the repository and set up your environment.
2. Review the [README.md](../README.md) and documentation links for project context.
3. Check the [TODO.md](TODO.md) for inspiration.
